# ES Module语法详解

目前市面上大多数浏览器都已支持ES Module语法，只需要在html中添加type为module的script标签即可

  <script type='module'>
    /**
     * 1. ESM自动采用严格模式，全局作用域下的this为undefined
     * 2. 每一个es module标签都是一个独立的作用域
     * 3. 外部资源的ESM是通过CORS的方式请求的
     * 4. ESM中的script标签会自动延迟执行脚本，在相当于给script标签添加了defer属性
     * 
    */
  </script>

# ES Module使用

每一个模块都是一个独立的作用域，内部的变量在外部无法被直接访问，若要使用模块内的成员，需要使用export关键字将变量导出，在外部使用import将变量导入。

export关键字的用法：

  1. 直接在变量前面加上export
  2. 在模块末尾用一对{ }来批量导出
  3. 在导出时可以给对应变量重命名，这样在接收时就只能使用重命名后的变量名
  4. 可以使用default关键字，导出一个默认变量
// 在module.js中：

  export let name = 'foo' // 用法1
  let age = 20
  function bar() {}
  class Person{}

  export { // 用法2，批量导出
    age, 
    bar as barName // 用法3，重命名
  }

  export default Person // 用法4，默认导出

import的用法

  1. 在import导入成员时，需将批量导出的成员写在花括号内，同时严格按照导出模块规定的变量名来接收
  2. 默认导出的成员，可以以任意名字接收，见下文PersonClass
  3. 在type=module的script标签里，js后缀和‘./’不能省略

import PersonClass,{ name,age,barName } from './module.js'  

console.log(PersonClass,name,age,barName)

# 注意点

  • {}是固定写法，不是对象的新写法和解构赋值
  • 导出的是变量的引用，且不能修改
